No time. Internally equal to -1 seconds, but should result in the number being ignored. For example, in me.bristermitten.plumber.scheduling.Task if the period is NONE, then a task is scheduled as only running once, whereas a period of EMPTY may cause the event to repeatedly run as the delay between execution is 0 ticks
